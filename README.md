# Причесыватель презентаций — WB Техношкола

Статическая страница для GitHub Pages: принимает до 15 .pptx, пересобирает их
по дизайн-коду и отдаёт исправленные файлы + список замечаний по каждому.

Обработка идёт **целиком в браузере** (Pyodide = Python в WebAssembly), поэтому
бэкенд не нужен и файлы преподавателей не покидают их компьютер.

## Структура

```
index.html            — страница (только внешний вид)
app.js                — UI-логика (только внешний вид)
engine/
  engine.js           — загрузка Pyodide, контракт convert()
  engine.py           — ВСЯ механика конвертации
  assets/guidelines.pptx        — шаблон дизайн-кода (донорские слайды)
  assets/DejaVuSansMono*.ttf    — шрифт код-карточек
```

Контракт между страницей и механикой — две функции:
`RestyleEngine.init(onProgress)` и `RestyleEngine.convert(arrayBuffer) →
{pptx, remarks:[{slide, action, comment}]}`. Всё, что глубже, можно менять,
не трогая страницу.

## Деплой на GitHub Pages

1. Создайте репозиторий, положите содержимое этой папки в корень, запушьте.
2. Settings → Pages → Source: Deploy from a branch → `main` / `/ (root)`.
3. Страница появится на `https://<user>.github.io/<repo>/`.

Первая загрузка движка у пользователя — ~15–20 МБ (Pyodide с CDN), дальше кэш.
Конвертация одного файла — секунды.

## Что умеет движок (v1, без LLM)

- титульный слайд из фирменного шаблона;
- рестайл всех контентных слайдов: фон и заголовки по гайдам,
  Inter / JetBrains Mono, все цвета текста → #080040, ссылки → #FF067E;
- типографика: неразрывные пробелы (предлоги, тире, числа), single line spacing,
  уровни body text 20/14/10 pt;
- солвер наездов: подбор кегля, зажим ширины, подъём блоков;
- снятие буллетов у списков из 1–3 пунктов;
- код, набранный текстом → карточка в стиле carbon.now.sh;
- добавление обязательных слайдов «Вопросы» и «Спасибо»;
- замечания: только то, что требует внимания (заглушки, сложные композиции,
  возможные наезды, нераспознанные заголовки).

## Дорожная карта механики (engine.py, страница не меняется)

- LLM-классификация макетов (плитка, колонки, картинка с подписями) — как в
  ручном прототипе;
- vision-шаг: скриншоты кода → карточки, генерация «Плана урока»;
- подключение внешнего типографа (typograf.artlebedev.ru) вместо локальных правил;
- прунинг неиспользуемых медиа шаблона (уменьшит размер выходного файла).
